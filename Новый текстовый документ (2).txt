Модуль № 1
Разработка базы данных средствами СУБД
Продолжительность выполнения: 0 ч. 50 мин.

Задание
Компания занимается продажей компьютерных комплектующих: процессоры, видеокарты, материнские платы, оперативная память, блоки питания, накопители SSD/HDD.

При запуске приложения окно входа – первое, что видит пользователь. На ней пользователю предлагается ввести свой логин и пароль, взятый из базы данных, или есть возможность перейти на экран просмотра товаров (без фильтрации, сортировки, поиска) в роли гостя.

Только после удачной авторизации пользователь получает доступ к остальным модулям системы:

Авторизованный клиент – просмотр товаров (без фильтрации/сортировки/поиска);

Менеджер – просмотр товаров (с фильтрацией, сортировкой, поиском), просмотр заказов;

Администратор – просмотр (с фильтрацией/сортировкой/поиском) / добавление / редактирование / удаление товаров, просмотр / добавление / редактирование / удаление заказов.

База данных должна быть в 3НФ с обеспечением ссылочной целостности.
Создать ER-диаграмму в формате PDF (таблицы, связи, атрибуты, ключи).

Выполнить импорт начальных данных из файлов components.xlsx и suppliers_import.xlsx (структура описана в приложении к заданию).
Сохранить скрипт БД.

Модуль № 2
Разработка алгоритма и создание приложения
Продолжительность выполнения: 0 ч. 40 мин.

Задание
Разработать блок-схему алгоритма работы приложения по ГОСТ 19.701-90 (PDF).
Обеспечить единый стиль интерфейса согласно руководству. Установить иконку приложения и логотип компании.

Оформление кода: CamelCase (C#/Java) или snake_case (Python), не более одной команды в строке, комментарии только в неочевидных местах.

Подключить базу данных к приложению. Реализовать вывод списка товаров с фото (заглушка no-image.png), названием, категорией, описанием, производителем, поставщиком, ценой, единицей измерения, количеством на складе, скидкой.

Подсветка строк
Если количество на складе = 0 → строка светло-серая.

Если скидка ≥ 20% → фон строки #A569BD (фиолетовый).

Если скидка > 0 – основная цена перечёркнута красным, итоговая цена чёрная.

Отладка и документация
Выполнить отладку, исключить аварийные завершения.
Создать .docx со скриншотами работы системы.

Модуль № 3
Сортировка, фильтрация, поиск и CRUD товаров
Продолжительность выполнения: 1 ч. 30 мин.

Навигация и обработка ошибок
Перемещение между окнами (кнопка «Назад»).

Информативные сообщения об ошибках (с пиктограммами и действиями по исправлению).

Визуальные подсказки при вводе.

Поиск / фильтр / сортировка (менеджер и администратор)
Поиск – по всем текстовым полям (название, описание, категория, производитель, поставщик) одновременно.

Сортировка – по цене и по остатку на складе (возрастание/убывание).

Фильтр по скидке (выпадающий список):

Все диапазоны

0% – 9,99%

10% – 19,99%

20% и более

Условия:
Поиск, фильтрация и сортировка работают в реальном времени (без кнопки «Применить»).
Параметры фильтрации и поиска применяются совместно.
Выбранная сортировка сохраняется при изменении поиска/фильтра.

CRUD товаров (только администратор)
Форма добавления/редактирования: фото (предпросмотр), название, категория (список), описание, производитель, поставщик (список), цена (≥0, до 2 знаков), единица измерения, количество (≥0), скидка (0–100).

ID – скрыт при добавлении, только для чтения при редактировании.

Изображения

Сохраняются в папку uploads/products.

В БД – относительный путь.

При замене – старое удаляется.

Ограничение: 300×200 пикселей (если больше – масштабировать или отклонить).

Запрет на открытие двух и более окон редактирования одновременно (клиентская блокировка).

Удаление – только если товар не используется в заказах.

После любых изменений – обновление списка с сохранением текущих параметров поиска/фильтра/сортировки.

Модуль № 4
Работа с заказами (Менеджер и Администратор)
Продолжительность выполнения: 1 ч. 00 мин.

Интерфейс заказов
Добавить кнопку «Заказы» для менеджера и администратора.
Список заказов отображает: артикул заказа, статус, адрес пункта выдачи, дата заказа, дата выдачи, ФИО клиента, итоговая сумма.

Управление заказами (только администратор)
Добавление / редактирование в отдельной форме.
Поля: артикул (уникальный), статус («Новый», «В обработке», «Завершён»), пункт выдачи (список), дата заказа, дата выдачи (может быть пустой).

Удаление заказа.

После сохранения/удаления – обновление списка заказов.

Переходы: по кнопке «Добавить заказ» → пустая форма; по строке заказа → форма редактирования.
